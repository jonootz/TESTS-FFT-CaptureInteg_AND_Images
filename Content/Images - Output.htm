Images - Output

  Images - Output
 

Support in Web Outputs:
 

Support in HTML5 Outputs:
  Verify that the entire <img> tag and all of its properties are removed when conditionalized out.
  Verify that the entire <img> tag and all of its properties are unbound when conditionalized out with the "Unbind" behavior.
  Verify that applied styles are in outputs
  Image Types Supported: *.bmp; *.gif; *.jpg; *.jpeg; *.png; *.tif; *.tiff; *.swf; *.hdp; *.wdp; *.emf; *.wmf; *.xaml; *.xps; *.exps; *.svg; *.ps; *.eps; *.pdf
  Verify the filters include all supported image file types when Importing an image.
  Using the Images test project listed at the top, check each of these image types.
  Some of the types are not supported fully, but we need to make sure we don’t crash or error out when working with these types of images.
  Verify all target frame settings for images for all web output types as well as topic previews.
  Verify that all supported image file types are supported in all outputs.
  Verify that *.ps, *.eps and *.pdf images are converted to *.png files. (Requires Ghostscipt and Java. See Compile Errors section below)
  Test that ALL combinations of the following Advanced Target options and verify that in all cases images work properly:
  Generate Resized Images
  Generate "web-safe" images
  Replace reserved characters with underscores
  Use lowercase filenames
  Test with "Responsive output" on and off (HTML5 Skin editor).
  Verify that all web supported images work correctly in "Slideshow".
  * Test ALL combinations.Do not simply test all options together or each option individually. There are 15 combinations.
  ** Be sure to setup test cases that will use the option. For example, use an image file called "My Image.svg" so that in the output it will become "my_image.jpg". The point is to test that the image link still works in the output after being processed by the target.
  tiff/tif files are not supported by IE 7 or 8 but they are in IE 9 and above (for tif/tiff files, enable generate web safe images in the advanced tab of your target to be able to view them in web outputs)
  Verify that images linked to on the web are dynamic by changing the image on the web.
  Verify that inserted pdf images will be generated as web-safe images even when pages other than page one of the PDF was used.

Support in DITA Output:
  For DITA output, the <img> tag will become a <image href="Resources/Images/Ashlighting.jpg" /> tag showing the relative path to the image. The image files themselves should simply be copied to the output folder.
  Note: For *.svg and *.pdf, the image file remains as is and not converted to a web-safe image type.
 

Requirements:
 

Ghost Script required for *.ps ,*.eps and *.pdf file types. Verify that the link works correctly.
   
 

Java is required for .svg file types. Verify that the link points to the Java download center.
